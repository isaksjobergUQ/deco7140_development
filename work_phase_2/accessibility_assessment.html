Accessibility Assessment

Summary of 13 WCAG 2.2 success criteria we’re focusing on, with actions tailored to our client and users.

Which Criteria Are Most Relevant (and Why)

  • Priya (mobile/low data): 1.1.1, 1.4.3, 1.4.4, 3.2.3, 3.3.2 – lightweight, readable, consistent, minimal fields.
  • Hana (low digital literacy): 1.4.1, 2.4.7, 3.1.5, 3.2.4, 3.3.1 – clear cues, visible focus, plain language, consistent UI, friendly errors.
  • Ava (local participant): 1.3.2, 1.4.3, 3.2.3, 3.3.4 – predictable order, contrast, consistent nav, easy recovery.

Perceivable

1.1.1 Alternative Text for Non-text Content

Definition: Provide text alternatives for images/icons so content is accessible and usable on low bandwidth.

  • Meaningful alt for informative images; alt="" for decorative.
  • Prefer real text to text-in-images; captions/transcripts for media.
1.3.2 Meaningful Sequence

Definition: Reading order matches visual order.

  • Semantic structure; logical DOM order; avoid CSS re-order traps.
1.4.1 Avoid Color-only Cues

Definition: Don’t rely on colour alone for meaning.

  • Add labels/icons; underline links; text explanations for statuses/errors.
1.4.3 Perceivable Text Contrast

Definition: Ensure sufficient text contrast.

  • Body 4.5:1, large text 3:1; avoid photo-overlays; use design tokens.
1.4.4 Resize Text without Loss

Definition: Zoom to 200% without loss of content/function.

  • Use rem/em; allow wrapping; avoid fixed-height containers.

Operable

2.1.1 Keyboard Navigability

Definition: All functionality works via keyboard.

  • TAB/ENTER/SPACE support; no focus traps; keyboard events.
2.4.7 Focus Visible

Definition: Focus indicator is always visible.

  • High-contrast outlines; consistent focus order; don’t remove outlines.

Understandable

3.1.5 Reading Level

Definition: Provide lower-secondary reading level or simplified summaries.

  • Plain headings; short sentences; glossary for visa/health terms.
3.2.3 Consistent Navigation

Definition: Navigation is consistent across pages.

  • Same menu order; same mobile menu; consistent breadcrumbs.
3.2.4 Consistent Identification

Definition: Same components look/behave the same.

  • Standardise cards/buttons/icons; consistent action labels.
3.3.1 Error Identification (& Suggestions)

Definition: Errors are clearly shown and described.

  • Inline text next to field; icon + text; aria-live="polite".
3.3.2 Labels (& Instructions)

Definition: Inputs have clear labels/instructions.

  • <label for>; helper text; mark optional vs required; minimise fields.
3.3.4 Error Prevention and Recovery

Definition: Prevent errors; easy recovery.

  • Confirm critical actions; undo where possible; don’t wipe forms on error.
